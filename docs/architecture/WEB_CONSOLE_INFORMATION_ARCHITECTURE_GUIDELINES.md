# Web Console 信息架构指引

## 各页面首屏优先级

- `新建翻译`：页面标题、YouTube 链接、说话人数、音色参数、创建按钮、当前是否已有进行中的任务
- `当前任务`：当前状态、当前阶段、当前一句进展、是否等待审核、主操作按钮、项目详情 / 结果入口
- `我的项目`：项目标题、状态、最近更新时间、进入项目详情
- `项目详情`：视频标题、当前状态、结果摘要、结果下载入口、少量关键日志
- `三类审核`：审核类型、待处理数量、当前必须完成的操作、主按钮、分页
- `设置说明`：当前版本说明、当前支持范围、当前限制说明、默认项说明

## 各页面主区块顺序

- `新建翻译`：标题与说明 -> 当前任务状态 -> 任务输入表单 -> 创建前说明
- `当前任务`：标题与状态 -> 审核强提示（如有） -> 阶段进度 / 当前结果 -> 次级输入信息与关键日志
- `我的项目`：页头统计 -> 项目列表
- `项目详情`：标题与状态 -> 审核提示（如有） -> 结果下载 -> 结果摘要 -> 关键日志 -> 次级项目信息
- `三类审核`：顶部主操作区 -> 待处理内容列表 -> 侧边审核概览 / 次级信息
- `设置说明`：版本说明 -> 支持范围 -> 限制说明 -> 默认项说明

## 次级 / 折叠信息

- 文件路径、内部结果、采样细节放入次级区或折叠区
- 创建时间、更新时间、任务输入参数保留为辅助理解信息，不抢主操作区
- 日志默认只显示少量关键记录

## 用户可见入口层级

- 默认主入口：新建翻译、当前任务、我的项目、项目详情、三类审核、设置说明
- 审核等待时优先在当前任务和项目详情中给出唯一主按钮
- 用户主视图不再显式展示旧 Web UI 或旧版审核页入口

## 旧 Web UI 不再显式展示的原因

- 当前目标是让单用户主流程形成统一前端心智
- 旧版入口会打断“当前在哪一步、下一步做什么”的理解
- 底层兼容链路可暂时保留，但不再作为最终用户可见入口
