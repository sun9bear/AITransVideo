## 4. 你的任务（P3e-3b/c + P3e-4，**必须按序**）

### ⚠️ 钱-关键耦合（最重要，别违反）
**「跳分钟点」绝不能脱离「3min teaser」单独做**——若 create 跳了分钟 reserve 但 pipeline 仍跑**完整视频**，就是**免费完整任务**（漏收全部分钟点 ≫ 克隆 600）。**所以顺序是 teaser 先、跳分钟后。一个半成品 P3e-3 本身就是钱-bug。**

### P3e-3b：3min teaser 裁剪（consumer 侧，仍扣分钟=安全）
- 找到匿名预览的 3 分钟 teaser 裁剪机制（候选：`gateway/anonymous_preview_probe.py` / `src/services/anonymous_preview_intake.py` / output publish 路径；用 understand workflow 定位），扩展到 `job_smart_preview`（pipeline 已读，见 process.py:2799 附近 `job_smart_preview`）。
- 这步仍扣分钟（无漏收），只限制**输出**到 3 分钟。

### P3e-3c-1：跳分钟（**必须在 teaser 之后**）
- gateway create：对 smart preview job（`request_data.get("preview_mode") is True` + flag）**跳过分钟 reserve**（job_intercept.py 的 `reserve_credits_or_raise` 那段，约 L2024）。只保留已有的 600 克隆 reserve。
- 跳分钟结算**自然达成**：create 没 reserve 分钟 → `settle_job_quota` / `settle_job_credit_ledger` 天然 no-op，**无需改 settlement**。
- ⚠️ 这步动的是**刚硬化过的 reserve 块**（P3e-2b 经 CodeX 3 轮修 7 类钱-bug），改动后必须重跑 `tests/test_p3e_create_reserve.py` + CodeX 复核别回归那 7 类。

### P3e-3c-2：preview→正式 server 复用契约
- 新 gateway 端点 / create 路径：用户从预览转完整时**前端只传 `preview_job_id`**，server 校验：该 preview 属同一登录用户 + 有 captured `clone_billing_event` + voice 已入 user_voices → server 端取回 voice_id + 原视频引用复用。**绝不信任前端传任意 voice_id**（防越权）。

### P3e-4：前端 + 反滥用
- `frontend-next/`：smart 提交前**预扣弹窗**（600 + 余额 `getMyCredits` + `clone_skipped_reason` 降级提示：insufficient_credits/voice_library_full/clone_disabled）；`smart_consent.auto_voice_clone` 改用户确认驱动（现 jobs.ts 硬编 true）；`preview_mode` 入口；`smart_preview_clone_enabled` admin 旋钮去占位（现灰显"预留"）。

## 5. 一直用的流程（务必沿用）

**每一个子切片都走：understand → TDD → implement → 验证 → 对抗性复核 → CodeX → 修 → commit。**

1. **understand**（动敏感/不熟代码前）：用 `Workflow` 工具起并行 Explore reader 映射子系统（agents 读 **worktree 绝对路径**，因 main repo 的 gateway 是 stale）。脚本是纯 JS，**别用 `[:1]`/`{ascii}` token**（template literal 会 parse 错；用字符串拼接传 prompt）。
2. **TDD**：先写测试。钱-correctness 测试优先（断言：双扣/双退/漏收/漏扣/inert/fail-safe）。
3. **implement**：小步、additive、默认 inert。
4. **验证**：`python -m pytest <files> -q`。gateway 测试需 `AVT_INTERNAL_API_KEY=test_key_at_least_16_chars_long_xxxx python -m pytest ...`（否则 build_job_api_server SystemExit）。
5. **对抗性复核**：用 `Workflow` 起 3-4 个 skeptic（各从不同 lens 试图 refute 钱-不变量，default refuted=true）。已抓出真 bug 多次。
6. **CodeX 外审**：`codex exec --sandbox read-only --cd D:/Claude/worktrees/anon-clone-enable "$(cat /tmp/prompt.txt)"`（prompt 写文件避 shell 转义）。发现问题修了再让 CodeX 复核。
7. **commit**：显式 pathspec + `git commit -F <msgfile>`（中文 message，结尾 `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>` 由全局配置处理/或省略）。
