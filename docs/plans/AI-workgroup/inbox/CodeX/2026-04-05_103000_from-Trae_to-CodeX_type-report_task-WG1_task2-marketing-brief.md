# Task 2 营销页信息架构、文案骨架与 Stitch Prompt

## 1. 当前问题判断

*   **为什么当前前端结构还不能直接承接营销页**：
    当前 `frontend-next/src/app/layout.tsx` 在根节点强制包裹了 `<AppShell>`，这意味着所有路由（包括 `/`）都会渲染工作台的侧边栏和顶部导航。营销页需要宽屏、无干扰的沉浸式布局，当前的结构在物理上无法兼容。
*   **Task 2 对当前项目的价值**：
    Task 2 是项目从“内部工具”走向“商业化 SaaS”的门面。它解决了“用户来了看什么”和“如何引导转化”的问题，是建立信任、展示核心卖点（如 DSP 级对齐、增量计费）并引导用户进入 Trial 漏斗的关键。
*   **本轮先做 vs 必须等 Claude Code 落地**：
    *   **本轮先做**：信息架构设计、文案撰写、Stitch 视觉引导 Prompt。
    *   **必须等 Claude Code 落地**：Task 1 的路由组拆分（`(marketing)` 目录的创建）、`SessionProvider` 的实现（用于控制 CTA 按钮状态）、以及最终 React 组件的代码编写。

---

## 2. 首页信息架构 (`/(marketing)/page.tsx`)

*   **Section 1: Hero 区 (首屏)**
    *   **目标**：3秒内让用户知道我们做什么，并提供最直接的转化入口。
    *   **一句话文案方向**：专注长视频的 AI 翻译配音工作台，精准对齐，直出剪映草稿。
    *   **CTA**：主按钮（免费开始试用 / 进入工作台），副按钮（查看定价）。
*   **Section 2: 核心痛点与解决方案 (Features)**
    *   **目标**：打消专业用户的顾虑，展示技术壁垒。
    *   **一句话文案方向**：告别口型错位与生硬机翻，专为专业创作者打造。
    *   **模块划分**：
        1.  DSP 级精准对齐（非简单 LLM 猜测）。
        2.  工程化输出（支持导出剪映草稿，保留二次编辑能力）。
        3.  增量重生成（改一句台词，只扣一句的钱）。
*   **Section 3: 工作流展示 (How it works)**
    *   **目标**：展示产品界面的专业性（可使用当前工作台的截图或 Mockup）。
    *   **一句话文案方向**：从导入到导出，掌控每一个细节。
*   **Section 4: Final CTA (底部转化)**
    *   **目标**：页面底部的最后一次转化拦截。
    *   **一句话文案方向**：准备好提升本地化效率了吗？
    *   **CTA**：主按钮（立即领取 7 天试用）。

## 3. 定价页信息架构 (`/(marketing)/pricing/page.tsx`)

*   **页面模块顺序**：
    1.  Header (简短有力的标题)。
    2.  Trial Banner (横幅，突出试用权益)。
    3.  Pricing Grid (套餐对比卡片)。
    4.  FAQ (常见问题解答)。
*   **套餐卡片区组织**：
    *   分为三列：Free、Plus、Pro。
    *   **高亮 Plus** 作为推荐套餐。
    *   **核心卖点区**：在 Plus 和 Pro 卡片下方，单独开辟一个视觉区块，强调“增量重生成按增量计费”。
*   **Trial 说明区组织**：
    *   放置在价格卡片上方，作为视觉焦点。
    *   强调“无需绑卡”、“包含 [待锁定] 分钟源视频额度”。
*   **FAQ 应回答的问题**：
    1.  试用期结束后会怎样？
    2.  “增量计费”具体是怎么算的？
    3.  支持哪些导出格式？（强调剪映草稿）
    4.  如何计算视频时长？

---

## 4. 试用页信息架构 (`/(marketing)/trial/page.tsx`)

*   **试用页目标**：承接未登录用户的转化意图，明确告知试用权益，并引导至注册/登录流程。
*   **用户最关心的 3-5 个问题**：
    1.  我能免费用多久？（[待锁定] 天）
    2.  我能处理多长的视频？（[待锁定] 分钟）
    3.  试用版有功能阉割吗？（明确告知可体验 Studio 模式）
    4.  需要绑定信用卡吗？（不需要）
*   **页面应该突出哪些信息**：
    *   “零风险体验”、“无需绑卡”。
    *   清晰的权益列表（天数、分钟数、可用模式）。
*   **不应承诺哪些内容**：
    *   不承诺无限量使用。
    *   不承诺提供企业级 SLA 或专属客服。

*   **Trial 页主标题草案**：
    *   **主标题**：开启您的专业本地化之旅

*   **CTA 文案候选**：
    1.  免费开始试用
    2.  领取 [待锁定] 天试用
    3.  进入工作台
    4.  查看完整定价

## 6. Stitch Prompt 初稿

```text
# 角色与目标
你是一个顶级的 SaaS 网页设计师。请为 AIVideoTrans（一个专业的 AI 视频翻译配音工作台）设计营销层页面（Marketing Pages）。

# 设计范围
本次只需要设计以下 3 个页面，请使用 Tailwind CSS v4 和 shadcn/ui 风格：
1. 首页 (Home Page)
2. 定价页 (Pricing Page)
3. 试用引导页 (Trial Landing Page)

# 严格约束（非常重要）
- 绝对不要生成工作台内部的业务页面（如 Review、Workspace、Job 详情页）。
- 绝对不要生成 Billing 控制台或支付表单。
- 页面必须是宽屏的营销布局，不要带侧边栏。
- 保持设计风格现代、专业、克制（Dark mode 优先，深蓝/灰黑色调）。

# 页面内容要求
## 1. 首页
- Hero 区：大标题“重塑视频本地化工作流”，副标题强调“DSP 级精准对齐，直出剪映草稿”。包含两个按钮：“免费开始试用”和“查看定价”。
- Features 区：采用 3 列网格，分别展示“精准对齐”、“工程化输出(剪映)”、“增量计费”。
- CTA 区：底部大色块引导注册。

## 2. 定价页
- 顶部 Banner：醒目提示“领取 7 天 Plus 试用，无需绑卡”。
- Pricing Grid：三列卡片（Free, Plus, Pro）。高亮 Plus 套餐。
- 特别强调区：在卡片下方用一个横向 Banner 突出“独创增量计费：改一句台词，只扣一句的钱”。
- FAQ 区：手风琴折叠样式。

## 3. 试用引导页
- 极简的落地页风格，左侧是权益说明（7天试用，20分钟额度，体验 Studio 模式，无需绑卡），右侧预留一个表单占位符（用于后续接入手机号登录）。
```
