# Report: Task 2 Marketing Implementation Review (WG4)

## 1. 总体判断

- **当前 T2 marketing 实现整体非常优秀，完全可接受**。代码结构清晰，组件拆分合理，且严格遵守了不触碰后端逻辑和未冻结事实的边界。
- **与 `DESIGN.md` 的对齐程度极高**：
  - 成功摆脱了“紫色 AI 模板腔”，采用了 `deep blue / steel cyan` 的专业色调。
  - `dark-capable` 和 `contrast-led` 的分层策略执行到位（Hero 用深色，阅读区用浅色/中性色）。
  - 中文优先原则在排版和文案上体现得非常自然，没有生硬的英文直译腔，Trust Cues（如“无需绑卡”）放置得恰到好处。

## 2. 应立即修的项

当前实现非常扎实，仅有一处文案需要立即修正以提升外部用户的信任感：

- **Trial 页未冻结状态的文案过于“内部视角”**：
  - **位置**：`components/marketing/trial-details.tsx` 中 `frozen === false` 时的渲染分支。
  - **当前文案**：“试用的具体天数与额度仍在最终确认中，以正式发放时的数据为准。”
  - **问题**：这听起来像内部测试版的占位符，会让外部访客觉得产品“还没准备好”，降低转化意愿。
  - **修改建议**：改为更自然的用户视角文案，例如：“**注册后即可查看并领取您的专属试用额度。**”（既不承诺具体数字，又保持了对外的专业度和引导性）。

## 3. 可后置 polish 的项

- **Workflow 区域的视觉丰富度**：目前 `workflow-showcase.tsx` 只有四个文字卡片。后续有了真实的产品截图或动图后，可以加入视觉素材，打破纯文本的单调。
- **Pricing 卡片的利益点微调**：目前 `planBenefits` 里的文案（如“单次视频最长 X 分钟”）非常直白，后续可以根据实际销售策略，在前面加上一些定性的价值描述（例如“超长处理：单次视频最长 X 分钟”）。
- **Footer 的丰富度**：目前 Footer 比较简单，后续可以加入更多合规链接（隐私政策、服务条款）和社交媒体入口。

## 4. 三页逐页建议

### 首页 `/`
- **保留**：Hero 区的“精准对齐，直出剪映草稿”非常抓人，Features 和 Workflow 的节奏很好，整体信息架构顺畅。
- **调整**：无须立即调整。
- **暂缓**：加入真实的视频 Demo 或产品界面截图（等待后续设计资产就位）。

### 定价页 `/pricing`
- **保留**：三档卡片的清晰对比，Plus 的高亮处理，以及 Trial Banner 作为转化入口而非第四个套餐的设计。优雅降级（Error 态）处理得很好。
- **调整**：无须立即调整。
- **暂缓**：具体价格数字和额度的最终确认（等待 Gateway 真相源冻结）。

### Trial 页 `/trial`
- **保留**：左侧定性权益说明（体验完整工作流、项目安全保留、无需绑卡）非常打动中文用户，消除了核心顾虑。
- **调整**：将 `frozen === false` 时的占位文案从“仍在最终确认中”改为“注册后即可查看并领取您的专属试用额度”。
- **暂缓**：右侧实际的注册/登录表单接入（属于 Task 3 范围）。

## 5. 不建议做的事

- **不要为了“好看”在 Pricing 和 FAQ 区域强加深色背景**：目前的浅色/中性背景非常适合中文长文本阅读，不要为了追求全站统一的“暗黑高级感”而牺牲可读性。
- **不要在 Trial 页强加复杂的动效**：Trial 页是转化漏斗的底端，目前的极简风格有助于降低认知摩擦，不要引入会分散注意力的视觉元素。
- **不要把 Marketing 的 Utility Classes 带入 App**：目前的 `.marketing-hero-surface` 等类名隔离做得很好，绝对不要在后续开发中把它们用到工作台（Workspace）或控制台（Admin）中。

- **文案微调**：在 `trial-details.tsx` 中，将 `frozen === false` 时的提示文案从“试用的具体天数与额度仍在最终确认中…”修改为更面向用户的“**注册后即可查看并领取您的专属试用额度。**”
- **放行确认**：除上述微调外，当前 T2 代码已完全达到预期，无需进行大的结构或视觉重构，微调后即可视为 T2 阶段圆满完成。
