# Web Console MVP 第一批验收说明

## 当前状态

首批 3 页已经达到“单用户 Web Console MVP 第一批可用态”，可作为当前阶段的冻结基线。

本批次覆盖的真实可人工试用链路为：

- 新建翻译
- 当前任务
- 项目详情
- 白名单下载
- 从新前端跳转到旧 Web UI 处理 review

## 本轮完成内容

- 新前端可以基于真实 YouTube URL 创建任务。
- 当前任务页可以显示 active job；没有 active job 时回退显示最近任务。
- 项目详情页可以按 `jobId` 正常进入，不再 404。
- 项目详情页可以显示结果摘要和白名单下载入口。
- review 阶段的假可用“继续流程”按钮已移除。
- 新前端在 review 阶段会给出真实处理入口，并跳转到旧 Web UI。
- 已完成一次真实“确认并继续”人工闭环验收，且新前端回看时能反映后续状态变化。

## 首批 3 页当前真实能力

- 单用户、单活跃任务流程
- 仅支持 `youtube_url` 提交
- 当前任务轮询、状态、阶段、日志可见
- 项目详情按 `jobId` 访问
- 白名单 stable key 下载
- review 阶段跳转旧 Web UI 处理

## 当前明确边界

- 不包含第二批页面
- 不做“我的项目”
- 不做设置页
- 不做 internal 页面
- 不做登录/注册体系
- 不做数据库
- 不做多用户
- 不做商业化页面
- 不做 failed resume UI
- 不在新前端内重建完整 review 中心

## 部署注意事项

- 旧 Web UI 需要公网兼容反代：`/api/* -> 127.0.0.1:8876`
- 旧 Web UI 必须跟随当前 active review job，不能停留在历史旧任务上
