# WEB_CONSOLE_PHASE2_LIGHT_PAGES_SCOPE.md

## 1. 本阶段唯一目标

本阶段只有一个目标：

在当前已冻结的单用户 Web Console 基线之上，补齐第二批页面的轻量版，使连续使用更顺手。

当前阶段只围绕以下范围展开：

- 我的项目（轻量版）
- 设置（极简版）
- `internal` 仅保留弱入口，不作为主开发重点

## 2. 为什么现在优先做第二批页面（轻量版）

- 第一批主链路页面已经可用，`speaker_review` 与 `translation_review` 也已收口并冻结
- 当前更合理的下一步，是补齐单用户日常回看与说明型页面，而不是继续扩张 review 能力
- `voice_review` 仍属于条件性出现的 review 能力，不应在当前阶段抢占第二批页面范围
- 轻量版第二批页面可以提升连续使用体验，但不会把项目带向多用户或商业化阶段

## 3. 本阶段必须做

- 新增“我的项目（轻量版）”
  - 历史项目列表
  - 状态
  - 最近更新时间
  - 进入项目详情
- 新增“设置（极简版）”
  - 当前版本说明
  - 当前稳定基线说明
  - 当前限制说明
  - 少量默认项说明
- `internal`
  - 只作为弱入口保留
  - 本阶段不作为重点页面族推进
- 如要把 TTS 默认模型说明或选择纳入“设置”，必须先证明它仍属于极简说明范围，而不是配置中心扩张

## 4. 本阶段明确不做

- 不做 `voice_review` 原生前端化
- 不做登录 / 注册
- 不做数据库
- 不做多用户
- 不做商业化页面
- 不做 failed resume UI
- 不做 review 系统扩张
- 不做大范围视觉重构
- 不做复杂搜索 / 筛选体系
- 不做批量操作
- 不做完整配置中心
- 不做 provider key 明文编辑

## 5. 验收标准

- “我的项目（轻量版）”可以稳定展示历史项目列表、状态、最近更新时间，并能进入项目详情
- “设置（极简版）”只承担说明与边界表达，不伪装成完整配置中心
- `internal` 仍保持弱入口状态，没有被扩张成主页面族
- 当前阶段没有混入 `voice_review` 原生化、多用户、商业化或复杂配置能力
- 第二批页面实现后，整体边界仍保持单用户、轻量、非产品化扩张
