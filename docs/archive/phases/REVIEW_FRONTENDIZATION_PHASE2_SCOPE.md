# REVIEW_FRONTENDIZATION_PHASE2_SCOPE.md

## 1. 本阶段唯一目标

本阶段只有一个目标：

继续消除新前端对旧 Web UI 的审核依赖，把当前仍需跳旧页面的审核链路尽量搬进新前端。

## 2. 当前真实测试暴露出的剩余旧页面依赖是什么

- 当前 `speaker_review` 与 `translation_review` 主链路已进入新前端
- 当前仍会跳旧页面的审核场景，首先是 `voice_review`
- `voice_review` 目前仍依赖旧 Web UI 的 `voice-library` 审核与继续链路
- 对于已原生前端化的 `speaker_review` / `translation_review`，旧 Web UI 仍保留为异常或契约不满足时的 fallback

## 3. 本阶段必须做

- 先盘点当前仍会跳旧页面的审核场景
- 明确这些场景分别属于什么类型，尤其确认是否主要集中在 `voice_review`
- 在不扩范围的前提下，确定本阶段最小需要原生前端化的剩余审核路径
- 保留旧 Web UI fallback，但继续压缩它在正常审核链路中的主路径地位
- 如有必要，只做少量不扩范围的排版修补，不把页面美化当主主题

## 4. 本阶段明确不做

- 不做大范围排版重构
- 不做第二批页面重做
- 不做登录 / 注册
- 不做数据库
- 不做多用户
- 不做商业化页面
- 不做大范围视觉升级
- 不做 review 系统重构

## 5. 验收标准

- 当前仍依赖旧 Web UI 的审核场景已被清楚盘点
- 已明确哪些剩余审核路径属于本阶段最小原生前端化范围
- 正常审核链路对旧 Web UI 的依赖进一步下降
- 小范围排版修补若发生，也没有扩张成新的 UI 优化阶段
- 本阶段推进后，范围仍保持单用户、轻量、非产品化扩张
