## 1. Delta 审核结论

在正式 `DESIGN.md` 落盘后，对 `WG1` 和 `WG2` 的建议进行复核，结论如下：

*   **WG1 中仍可保留的建议**：

    *   **Trial 页的定位**：作为承接转化意图的落地页，强调“零风险体验”和清晰的权益列表。
*   **WG2 中需要调整表述的建议**：

    *   WG1 中提供的具体文案（如“重塑视频本地化工作流”、“简单透明的定价，为专业生产力买单”）仅为**候选文案方向**，不能作为最终锁定的静态文本。
    *   WG1 中 Stitch Prompt 初稿里关于“深蓝/灰黑色调”的描述，应调整为 `DESIGN.md` 规定的 `deep blue`, `steel cyan`, `signal teal` 等专业中性色调。

## 2. Task 2 三页 marketing 建议收口

### 2.1 首页 (Home Page)
*   **保留**：Hero 区引导、核心痛点解决（DSP 对齐、剪映草稿、增量计费）、工作流展示。

    *   **视觉层级**：Hero 区的 Demo 展示必须是视觉焦点，采用深色背景（dark-capable）以突出多媒体内容。
    *   **CTA 文案**：收口到 `DESIGN.md` 推荐的直接中文表达，如 `免费开始使用`、`立即翻译视频`。
*   **暂缓/不写死**：Hero 区的具体 Slogan 需根据最终产品定位微调，不应在代码中硬编码为不可配置的静态字符串（如果未来有 CMS 或 i18n 计划，当前可暂用候选文案占位）。

### 2.2 定价页 (Pricing Page)
*   **保留**：三列卡片结构（Free, Plus, Pro）、高亮 Plus 套餐、FAQ 区。
*   **调整**：
    *   **视觉层级**：Pricing Grid 和 FAQ 区域必须保证高对比度，使用清晰、明快的背景色，避免在深色背景上阅读密集的中文套餐对比。

### 2.3 Trial 页 (Trial Landing Page)

    *   **文案基调**：必须清晰说明试用期结束后的状态（如“试用结束后不会自动扣费”），消除用户的隐性担忧。
*   **暂缓/不写死**：试用的具体天数（如 7 天）、赠送的额度（如 20 分钟），必须标记为 `待项目开发者确认`。

## 3. Billing / Admin / App 边界说明

为了防止后续开发将工作台做成营销页风格，特此明确 `(app)` / billing / admin 的设计边界：

*   **不应外溢的 Marketing 元素**：
    *   **大面积深色背景 (Dark-first Hero)**：工作台和控制台需要长时间操作，大面积深色容易导致视觉疲劳。
    *   **超大号排版 (Oversized Typography)**：营销页为了冲击力使用的巨大标题，在工作台中会严重破坏信息密度和操作效率。
    *   **戏剧性动效 (Dramatic Motion)**：营销页的滚动视差、元素飞入等动效，在工作台中会显得拖沓且干扰操作。
    *   **营销导向的文案 (Marketing Slogans)**：工作台需要的是“状态明确、操作清晰”的微文案，不需要煽动性的口号。
*   **应继承的 Foundations / Guardrails**：
    *   **色彩体系 (Color Direction)**：继承专业的中性基础色和冷静的强调色（如 steel cyan），保持品牌一致性。
    *   **中文优先排版 (Chinese-First Typography)**：继承易读的字体栈、合理的行高和字重规范。
    *   **间距与圆角 (Spacing & Shape)**：继承 8px 间距系统和适度的圆角规范，保持 UI 组件的统一感。
    *   **信任表达原则 (Trust Principles)**：在 Billing 页面继承清晰、无歧义的状态标签和费用说明。
